Research UI implementation specification — v1 Normative terms: MUST means required for v1; SHOULD means recommended unless engineering constraints prevent it. 1. Product boundary and invariants Research UI responsibility The Research UI owns: Captured-source inspection Normalization and extraction review Evidence anchoring Entity resolution Claim formation Contradiction review Comparisons and benchmark analysis Editorial review Versioned publication It does not own: Careful live browsing Login/session handling for target websites Interactive source exploration before capture Infrastructure and pipeline observability Those remain in Rebrowser and the separate metrics surface. Rebrowser integration The global Capture source action MUST: Call Rebrowser.LaunchCapture. Pass project_id, optional seed_url, optional source_id, and return_route. Open the returned Rebrowser session in a new tab. Receive a capture.committed backend event. Add the resulting capture to the Research Inbox. Deep-link the source workbench back to the exact Rebrowser session. The source workbench MUST expose: Open in Rebrowser Recapture Capture related source View capture session metadata Evidence MUST point to the frozen capture artifact, never to the mutable live page. Epistemic layers The frontend MUST visually and structurally distinguish: Captured source — immutable. Machine observation — OCR, transcript, entity suggestion, extracted fact, VL result. Human-curated evidence and claims — reviewed research objects. Publication snapshot — frozen approved output. A model observation cannot directly set a claim to accepted or canonical. 2. Actions that must emit review events Review events are required for: Accepting, correcting or rejecting a model observation Accepting or rejecting evidence Changing an evidence anchor after review Resolving an entity candidate Creating or merging canonical entities Accepting, rejecting, disputing or superseding claim assertions Selecting a preferred assertion for a scope Resolving or reopening a contradiction Approving taxonomy changes that affect existing knowledge Overriding a publication check Requesting publication changes Approving a publication snapshot Publishing or superseding a snapshot Ordinary navigation, search, filter changes and private draft typing do not emit review events. 31.
